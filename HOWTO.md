# HOWTO

이 문서는 이 프로그램을 **아주 쉽게 실행하는 방법**을 설명합니다.

어렵게 생각하지 않아도 됩니다.
그냥 **순서대로 하나씩** 따라 하면 됩니다.

이 설명은 **macOS** 기준으로 썼습니다.
지금 이 프로젝트 폴더가 이미 열려 있다고 가정합니다.

만약 이 프로그램을 **Vercel에 올리는 방법**이 궁금하다면,
[HOWTO-vercel.md](HOWTO-vercel.md)를 보면 됩니다.

---

## 0. 이 문서의 목표

우리가 하고 싶은 일은 딱 이것입니다.

1. 국회 API 키를 준비한다.
2. 프로그램이 쓸 준비를 한다.
3. 프로그램을 켠다.
4. 브라우저에서 `http://localhost:3000/newsletter` 를 연다.

---

## 1. 먼저 준비할 것

아래 3가지만 있으면 됩니다.

1. 인터넷
2. 열린국회정보 API 키
3. Node.js

### 1-1. API 키가 뭔가요?

이 프로그램이 국회 데이터를 읽으려면 **열린국회정보 API 키**가 필요합니다.

받는 방법:

1. 브라우저를 엽니다.
2. `열린국회정보`를 검색합니다.
3. 사이트에 회원가입하고 로그인합니다.
4. 마이페이지에서 **OPEN API 인증키**를 발급받습니다.
5. 발급받은 긴 문자열을 복사합니다.

이 키는 잠시 뒤 `.env` 파일에 붙여 넣을 것입니다.

### 1-2. Node.js가 뭔가요?

이 프로그램을 켜 주는 엔진 같은 것입니다.

확인 방법:

터미널에 아래를 입력하세요.

```bash
node -v
```

그리고 이것도 입력하세요.

```bash
npm -v
```

둘 다 숫자가 나오면 괜찮습니다.

예:

```bash
v22.15.0
10.9.2
```

만약 `command not found` 같은 글자가 나오면 Node.js가 아직 없는 것입니다.
그럴 때는 브라우저에서 `Node.js LTS 다운로드`를 검색해서 설치한 뒤 다시 돌아오면 됩니다.

## 4. 처음 한 번만 해야 하는 준비

이 부분은 **처음 한 번만** 하면 됩니다.

### 4-1. 필요한 꾸러미 설치

터미널에 이것을 입력하세요.

```bash
npm install
```

이 작업은 조금 시간이 걸릴 수 있습니다.
끝날 때까지 기다리세요.

### 4-2. 환경설정 파일 만들기

터미널에 이것을 입력하세요.

```bash
cp .env.example .env
```

이 명령은 예시 파일을 복사해서, 진짜 설정 파일인 `.env`를 만들어 줍니다.

### 4-3. `.env` 파일 열기

프로젝트 파일 목록에서 `.env` 파일을 클릭해서 엽니다.

그 안에는 여러 줄이 있을 수 있습니다.
우리는 **아래 3개가 가장 중요**합니다.

```env
ASSEMBLY_API_KEY=여기에_발급받은_API_키를_붙여넣기
MCP_TRANSPORT=http
MCP_PORT=3000
```

### 4-4. API 키 넣기

예를 들어 여러분의 API 키가 `abcd1234`라면 이렇게 씁니다.

```env
ASSEMBLY_API_KEY=abcd1234
MCP_TRANSPORT=http
MCP_PORT=3000
```

중요:

- `ASSEMBLY_API_KEY=` 오른쪽에 키를 넣습니다.
- 따옴표는 보통 넣지 않아도 됩니다.
- `MCP_TRANSPORT=http` 로 되어 있어야 브라우저 화면이 열립니다.
- `MCP_PORT=3000` 은 브라우저 주소 번호입니다.

파일을 저장하세요.

## 5. 프로그램 실행하기

이제 프로그램을 켤 차례입니다.

### 5-1. 빌드하기

터미널에 이것을 입력하세요.

```bash
npm run build
```

이 작업은 프로그램을 실행 가능한 형태로 준비하는 과정입니다.

### 5-2. 실행하기

터미널에 이것을 입력하세요.

```bash
npm start
```

그러면 프로그램이 켜집니다.

중요:

- 이 창은 **닫지 마세요**
- 이 창이 켜져 있어야 웹 화면도 작동합니다

## 6. 브라우저에서 열기

이제 브라우저를 엽니다.

주소창에 이것을 입력하세요.

```text
http://localhost:3000/newsletter
```

그러면 입법예고 뉴스레터 화면이 열립니다.

이제 여기서 할 수 있는 일:

- 키워드 검색
- 날짜 선택
- 검색 결과 보기
- HTML 미리보기
- HTML 저장
- Markdown 저장
- HWPX 저장

### 6-1. 화면 미리보기

처음 열면 대략 이런 느낌의 화면이 보입니다.

실제 색깔이나 모양은 조금 다를 수 있지만,
**큰 구조는 거의 이렇게 생각하면 됩니다.**

```text
+--------------------------------------------------------------+
| 입법예고 뉴스레터 스튜디오                                   |
| 키워드와 기간을 넣고 관련 법안을 찾는 화면                    |
+--------------------------------------------------------------+

+---------------------------+----------------------------------+
| 검색 영역                  | 상세 미리보기                    |
|                           |                                  |
| 키워드: [인공지능      ]  | 법안 제목                         |
| 발의 의원: [          ]   | 의안번호                          |
| 상임위:   [          ]    | 발의 의원                         |
| 기간: [1개월] [검색]      | 상임위                            |
|                           | 현재 단계                         |
+---------------------------+----------------------------------+

+--------------------------------------------------------------+
| 검색 결과 목록                                                  |
| [ ] 법안 A                                                     |
| [ ] 법안 B                                                     |
| [ ] 법안 C                                                     |
+--------------------------------------------------------------+

+--------------------------------------------------------------+
| 내보내기 및 저장                                                |
| HTML 미리보기 | HTML 저장 | Markdown 저장 | HWPX 저장         |
+--------------------------------------------------------------+
```

쉽게 말하면:

1. 왼쪽 위에서 검색 조건을 넣습니다.
2. 가운데에서 결과 목록을 봅니다.
3. 오른쪽에서 선택한 법안 내용을 미리 봅니다.
4. 아래에서 미리 보거나 파일로 저장합니다.

### 6-2. HTML 미리보기는 어디 있나요?

화면 아래쪽 `발송 및 저장` 영역에 버튼이 있습니다.

이 버튼들로 바로 확인할 수 있습니다.

- `선택 항목 HTML 미리보기`
- `전체 결과 HTML 미리보기`

버튼을 누르면 **문서가 어떻게 보일지 그대로** 미리 볼 수 있습니다.

쉽게 말하면:

- 저장하기 전에
- 문서가 어떻게 보일지
- 먼저 확인하는 기능입니다.

### 6-3. 어떤 버튼을 누르면 되나요?

가장 쉬운 사용 순서는 이렇습니다.

1. 키워드를 넣습니다.
2. `검색`을 누릅니다.
3. 보고 싶은 법안을 체크합니다.
4. `선택 항목 HTML 미리보기`를 누릅니다.
5. 마음에 들면 `선택 항목 Markdown 저장` 또는 `선택 항목 HWPX 저장`을 누릅니다.

전체 검색 결과를 한 번에 보고 싶다면:

1. 키워드를 넣습니다.
2. `검색`을 누릅니다.
3. 아무것도 체크하지 않아도 됩니다.
4. `전체 결과 HTML 미리보기`를 누릅니다.
5. 마음에 들면 `전체 결과 Markdown 저장` 또는 `전체 결과 HWPX 저장`을 누릅니다.

## 11. 아주 자주 생기는 문제

### 문제 1. `ASSEMBLY_API_KEY가 설정되지 않았습니다`

뜻:
API 키가 없거나, `.env`에 제대로 안 들어간 것입니다.

해결:

1. `.env` 파일을 엽니다.
2. `ASSEMBLY_API_KEY=` 줄을 찾습니다.
3. 오른쪽에 진짜 API 키가 들어 있는지 봅니다.
4. 저장합니다.
5. 서버를 껐다가 다시 켭니다.

### 문제 2. `npm` 또는 `node`를 못 찾는다고 나옴

뜻:
Node.js가 아직 설치되지 않았습니다.

해결:

1. 브라우저에서 `Node.js LTS 다운로드` 검색
2. 설치
3. 터미널을 닫았다가 다시 열기
4. 다시 실행

### 문제 3. 브라우저에서 페이지가 안 열림

뜻:
서버가 안 켜졌거나, 다른 프로그램이 같은 포트를 쓰고 있을 수 있습니다.

해결:

1. 터미널 창이 아직 켜져 있는지 확인
2. `npm start`를 다시 실행
3. 그래도 안 되면 `.env`에서 포트를 바꿔 보기

예:

```env
MCP_PORT=3001
```

그다음 다시 실행하고 브라우저에서 이렇게 엽니다.

```text
http://localhost:3001/newsletter
```

### 문제 4. HWPX 파일이 저장되지 않음

뜻:
브라우저가 다운로드를 막고 있거나, 선택한 법안이 없을 수 있습니다.

해결:

1. `선택 항목 HWPX 저장`이면 법안을 1건 이상 체크했는지 확인
2. `전체 결과 HWPX 저장`이면 검색 결과가 실제로 있는지 확인
3. 브라우저 주소창 근처에서 다운로드 차단 알림이 없는지 확인
4. 서버 다시 실행

## 15. Vercel에 올리고 싶을 때

이 프로그램은 `Vercel`에도 올릴 수 있습니다.

하지만 기억해야 할 점이 있습니다.

- 화면 열기
- 검색
- 미리보기
- Markdown 저장
- HWPX 저장

는 Vercel에서 바로 쓸 수 있습니다.

반대로,

- 저장된 수신자
- 검색 preset
- 발송 로그
- 예약 발송

같은 것은 `저장소 설정`이 같이 있어야 더 안정적으로 동작합니다.

가장 쉬운 안내는 [HOWTO-vercel.md](HOWTO-vercel.md)를 보면 됩니다.

조금 더 기술적인 설명까지 같이 보고 싶다면 [VERCEL.md](VERCEL.md)도 보면 됩니다.

아주 짧게 말하면:

1. GitHub에 프로젝트를 올립니다.
2. Vercel에서 그 저장소를 가져옵니다.
3. `ASSEMBLY_API_KEY`를 환경 변수에 넣습니다.
4. 저장 기능까지 안정적으로 쓰려면 `NEWSLETTER_STORAGE_BACKEND=vercel-blob`과 `BLOB_READ_WRITE_TOKEN`도 넣습니다.
5. 배포 후 `https://내주소/newsletter`로 들어갑니다.

중요:

- Vercel에서는 저장 경로가 기본적으로 `/tmp/.newsletter-data`가 됩니다.
- 이 폴더는 영구 보관용이 아닙니다.
- 그래서 저장 데이터가 안 사라지게 하려면 `Vercel Blob` 설정을 함께 하는 편이 좋습니다.

배포가 끝난 뒤에는 `/newsletter` 화면에서 바로 확인할 수 있는 것도 있습니다.

- `실행 환경 상태`에서 API 키, 저장소, 다운로드 준비 상태를 볼 수 있습니다.
- `Vercel 환경 변수 보기` 버튼으로 넣어야 할 환경 변수 예시를 볼 수 있습니다.
- `Vercel 배포 점검 보기` 버튼으로 배포 직후 무엇부터 확인해야 하는지 체크리스트를 볼 수 있습니다.
